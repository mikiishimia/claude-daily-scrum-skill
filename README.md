# Claude Daily Scrum Skill

**Claude × Notion × Google Calendar × Gmail を使った、一人事業者向けのデイリースクラム自動化スキル**

毎朝の段取りから週次レビューまでをサポートします。

---

## これは何か

Claude のプロジェクト機能に登録することで、毎朝「おはよう」と送るだけで以下が自動実行されるスキルファイルです。

- Google Calendar から今日の予定を取得
- Notion のバックログを確認
- Gmail の重要未読メールを確認
- デイリーアクション（3〜5個）を提案
- Notion のデイリーアクションDBに直接書き込み

一人事業者・フリーランス・スタートアップ創業者を想定して設計されています。

|-----------|---------|---------|

## セットアップ方法

### 1. 前提条件

Claude に以下のコネクタを接続してください：

| コネクタ | 必須/任意 | 用途 |
|---------|---------|------|
| Google Calendar | **必須** | 予定の取得 |
| Gmail | **必須** | 重要メールの確認 |
| Notion | **必須** | バックログ・デイリーアクションの読み書き |
| Slack | 任意 | チームへの共有 |

> **注意：** 初回セッション開始時にミキがコネクタの接続状況を確認します。未接続の場合は該当機能がスキップされます。

### 2. Notion の準備

以下の2つのデータベースを Notion に作成します。

**① バックログDB**（プロジェクト管理）

| プロパティ名 | タイプ |
|------------|--------|
| プロジェクト名 | タイトル |
| ステータス | セレクト |
| 次のアクション | テキスト |
| 優先度 | セレクト（高/中/低） |
| 期日 | 日付 |

**② デイリーアクションDB**

| プロパティ名 | タイプ |
|------------|--------|
| アクション名 | タイトル |
| 完了 | チェックボックス |
| 実行日 | 日付 |
| 関連プロジェクト | リレーション（バックログDBへ） |
| カテゴリ | セレクト |
| 時間目安 | テキスト |

### 3. 設定ファイルの記入

`config.example.md` をコピーして `config.md` にリネームし、自分の情報を記入します。

```bash
cp config.example.md config.md
```

### 4. daily-log.md の作成

プロジェクトファイルとして `daily-log.md` を作成します。日次ふりかえり後にミキが更新用テキストを生成するので、それを貼り付けて管理します。

```markdown
# デイリーログサマリー

## 直近の持ち越しタスク
（初回は空でOK）

## 継続中プロジェクト状況
（初回は空でOK）

## 最終更新：YYYY-MM-DD
```

### 5. Claude プロジェクトへの登録

1. Claude.ai で新しいプロジェクトを作成
2. プロジェクトの「ファイル」に以下を追加：
   - `SKILL.md`（このリポジトリの本体）
   - `config.md`（あなたの設定を記入したもの）
   - `daily-log.md`（ふりかえりのサマリーログ）
3. プロジェクトの「カスタム指示」に以下を貼り付け：

```
このプロジェクトではSKILL.mdに定義されたスクラムマスター「ミキ」として動作してください。
個人設定はconfig.mdを、直近の文脈はdaily-log.mdを参照してください。
```

## 使い方

### 基本の流れ

```
おはよう
```
→ Google Calendar・Notion・Gmail が確認され、今日のアクション提案が返ってきます。

### スレッドの使い方

**同じスレッドを継続して使用してください。** 毎日新しいスレッドを作ると文脈が失われ、完了済みタスクを何度も提案されるなどの問題が起きます。

スレッドが重くなってきたら（目安：週次レビュー5〜7回）、ミキが移行を提案します。引き継ぎプロンプトを自動生成するので、新しいスレッドにそのまま貼り付けてください。

---

## カスタマイズ

`SKILL.md` の以下のセクションを編集することで動作を変更できます：

- **就業時間** — `config.md` の `WORK_HOURS` で設定
- **営業ドメインフィルタ** — `config.md` の `GMAIL_PRIORITY_DOMAINS` で設定
- **営業パイプラインステージ** — `config.md` の `PIPELINE_STAGES` で設定

---

## 既知の制限

- **Gmail** — メールの読み取り・検索のみ対応。アーカイブ・ラベル変更・削除はGmail上で直接操作してください
- **Notion** — config.md に DB ID を設定しないと読み書きできません
